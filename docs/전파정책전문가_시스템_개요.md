# 전파정책 전문가 AI — 시스템 개요

> SK텔레콤 Comm센터 기술정책팀 | 작성일: 2026-05-27  
> 목적: 시스템 현황 공유 및 외부 검토·개선 의견 수렴

---

## 1. 프로젝트 목적

SK텔레콤 Comm센터 기술정책팀의 전파정책 담당자가 일상 업무에서 반복적으로 수행하는 작업을 AI로 보조하는 시스템입니다.

**핵심 과제:**
- 전파법·고시·ITU-R 권고 등 85개 공개 문서를 지식 베이스로 구축하여, 조문 근거를 동반한 실무 자문을 제공
- 국내외 전파·통신 정책 뉴스를 매일 자동 수집·요약하여 모닝 브리핑 제공
- 기술 용어를 자동으로 감지하고 DB에 누적

**주요 제약:**
- 회사 기밀 자료 불포함 — 공개 법령, 고시, ITU-R 문서만 사용
- 답변은 "검토 초안·분석 의견" 수준, 공식 법률 해석이 아님을 명시
- 법령 조문은 반드시 업로드된 원문에서 확인 후 인용 (추정 생성 금지)

### 3-5. 자동화 파이프라인 (모닝 브리핑)

**플랫폼:** Claude Cowork 스케줄 작업 (매일 오전 자동 실행)

**실행 흐름 (9단계):**

```
STEP 1  18개 키워드로 WebSearch 실행
        → IT전문지(전자신문·ZDNet·디지털데일리 등) 6개 검색
        → 경제·종합지(연합뉴스·매일경제·한국경제 등) 5개 검색
        → 정부사이트(msit.go.kr·rra.go.kr·law.go.kr 등) 7개 검색

STEP 2  최근 7일 이내 기사만 필터링, 최대 20건 목록 작성

STEP 3  DB에서 기존 기술 용어 조회 (중복 방지)

STEP 4  신규 기술 용어 목록 작성 (DB 미등록 전문 용어만)

STEP 5  뉴스 INSERT (title 기준 중복 제외)
        → 미브리핑 기사(briefed_date IS NULL) 조회

STEP 6  신규 기술 용어 INSERT

STEP 7  브리핑 텍스트 작성
        (주요뉴스 / 주목포인트 / 새 기술 용어 / 저장 결과)

STEP 8  daily_briefings 테이블에 브리핑 저장 (날짜 기준 upsert)

STEP 9  브리핑 포함 기사에 briefed_date 기록
        (다음 날 중복 브리핑 방지)
```

**현재 수집 키워드 분류:**
- 5G/6G/주파수/전파 정책 (IT전문지·종합지)
- 위성통신(NTN/HAPS/LEO), IMT-2030, 공공 와이파이
- 정부 보도자료·고시 (msit.go.kr, rra.go.kr, law.go.kr)

## 7. 현재 한계 및 알려진 이슈

### 기능적 한계

1. **RAG 품질**: 현재 뉴스 컨텍스트는 제목·날짜만 주입 (본문 없음). AI가 기사 내용을 알 수 없어 "최근 동향" 언급이 피상적.

2. **법령 원문 접근**: 질문마다 AI가 지식 베이스(85개 파일)를 전체 참조하지 않고 RAG 청크 5개만 참조하므로, 조문이 누락될 수 있음.

3. **ITU-R 문서 미완성**: M.2150-3(5G), M.2160-0(6G) 등 최신 버전 6개 미업로드.

4. **자동 크롤링 미운영**: crawler.py는 완성됐지만 GitHub Actions 연동이 안 된 상태. 현재 모닝 브리핑은 Claude Cowork 웹 검색으로 대체 중.

5. **뉴스 수집 범위**: 7일 이내 기사만 대상으로 하여, 중요 배경 기사(1~4주 전)가 누락될 수 있음.

### 구조적 한계

6. **단일 사용자 구조**: Supabase Auth 미적용 상태로, 현재 복수 팀원이 사용하기 어려운 구조.

7. **지식 베이스 갱신**: 법령·고시 개정 시 수동으로 Claude Projects에 파일을 재업로드해야 함 (변경 자동 감지 미구현).

8. **모닝 브리핑 알림 채널**: 현재 Supabase에만 저장되고 팀 채널(Slack 등)로 발송되지 않음.

## 8. 개선 방향 (검토 의견 요청 항목)

다음 항목에 대한 피드백을 구합니다.

**A. 아키텍처 설계**
- 웹 대시보드를 단일 HTML 파일로 유지하는 것이 적절한가, 아니면 분리가 나은가?
- GitHub Pages가 아닌 다른 배포 방식(Cloudflare Pages, Vercel 등)이 더 적합한가?

**B. RAG 품질 개선**
- 현재 키워드 매칭 방식 RAG의 한계를 극복하기 위해 벡터 임베딩(pgvector) 도입이 필요한가?
- 법령 문서는 조항 단위로 청킹하는 것이 효과적인가?

**C. 지식 베이스 관리**
- 법령 개정 자동 감지·갱신 파이프라인을 어떻게 구성하면 좋은가?
