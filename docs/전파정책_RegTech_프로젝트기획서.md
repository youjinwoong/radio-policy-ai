# 전파정책 RegTech — 프로젝트 기획서

> SK텔레콤 Comm센터 기술정책팀 | 작성일: 2026-05-27
> 기존 "전파정책전문가" 시스템을 계승·발전시킨 신규 독립 프로젝트

---

## 1. 프로젝트 배경 및 목적

### 1-1. 기존 시스템의 한계 (팀장 피드백)

2026년 5월 팀장 검토 결과, 기존 "전파정책 나침반"은 다음 문제가 지적됐다.

| 문제 | 내용 |
|---|---|
| 차별성 부재 | "일반 Claude/GPT에 물어봐도 비슷한 답변" |
| 정보 피로도 | 매일 브리핑이 발송되지만 "오늘 뭘 해야 하는지"를 말하지 않음 |
| 느린 응답 | RAG 검색 단계가 많아 체감 속도 저하 |
| 단일 사용자 | 팀 전체 툴로 확산되지 않는 구조 |
| 명칭 불명확 | "전파정책 나침반"이 팀장에게 "와닿지 않는다"는 반응 |

### 1-2. 신규 프로젝트의 목표

> **"타 부서 챗봇·일반 GPT와 확실히 다른, 기술정책팀의 업무 생산성을 실제로 높이는 규제 기술(RegTech) 플랫폼"**

- 단순 정보 소비 → **규제 리스크 감지 + 팀 액션 아이템 자동 도출**
- 법령 Q&A → **KPI 연동 리스크 등급 + 연쇄 법령 교차 분석**
- 매일 브리핑 → **크리티컬 이슈만 즉시 알림, 나머지는 주간 요약**

## 2. 팀 페르소나

### 2-1. 기술정책팀 역할

SK텔레콤 Comm센터 내 기술 규제 대관 전담 조직으로, 정부·국회가 SKT에 불리한 규제를 만들기 전에 막고, 유리한 규제 환경을 선제적으로 조성하는 팀.

**담당 업무 영역**
- 이동통신 주파수: 할당, 재할당, 전파사용료, 주파수 경매, 인허가, 무선국, 품질평가
- 통신재난: 재난문자, 이동통신 재난, 재난 대응
- 기술 규제 이슈: CR 조직 내 기술 관련 이슈 전담

### 2-2. 2026년 KPI (AI 리스크 판단 기준)

| KPI | 비중 | 핵심 방어 목표 | 재무 규모 |
|---|---|---|---|
| 무선국 검사·전파사용료 제도개선 | **35%** | 예비전원 의무화 저지, 전파사용료 감면 확보 | ~수백억 원 절감 |
| 추가 주파수·재할당 대가 산정 | **35%** | 5G 추가 주파수 2027년 이후 지연, 재할당 대가 인상 저지 | 중장기 수익성 |
| 우호적 품질평가 정책 | **15%** | SKT 우호 평가 방식 확보, SA 시범평가 유지 | 투자 부담 최소화 |
| 통신재난·B2B 지원 | **15%** | 재난문자 ESG 성과, B2B 정책 지원 3건 이상 | B2B 수주 |

## 4. 핵심 기능 8가지

### 기능 1 — 뉴스·보도자료 기반 팀 액션 아이템 자동 도출

**개념:** 모닝 브리핑 수집 뉴스를 분석하여 "기술정책팀이 오늘 해야 할 실무 액션 리스트"를 강제 생성. "뉴스를 읽고 뭘 해야 하지?"를 고민하는 시간을 0으로 단축.

**출력 형식:**
```
[뉴스] 전파법 시행령 개정안 입법예고
  ↓
[팀 액션 아이템]
즉시 확인  | Diff 분석 실행 후 신구조문 대조
임원 보고  | 1p 보고서 초안 (공문 대응 모드)
유관부서   | 재무팀: 대가 증가 시나리오 분석 요청
산출물     | 정부 의견서 초안
일정 알림  | 의견제출 기한 등록 (~발표일+20일)
```

**KPI별 액션 템플릿:**

| 이슈 유형 | 즉시 확인 | 유관부서 협의 | 필요 산출물 |
|---|---|---|---|
| 주파수 재할당 | Diff 분석, 원문 확인 | 재무팀·네트워크전략팀 | 임원 보고서 + 의견서 |
| 예비전원 의무화 | 대상 지역·Site 범위 | 네트워크팀 (Site 현황) | 반대 논리 + 대안 제시 |
| 전파사용료 | 감면계수 변경 내용 | 재무팀 (절감액 산출) | 의견서 |
| 품질평가 | 평가 방식·일정 변경 | 네트워크팀 (커버리지 현황) | 우호 방식 의견서 |
| 통신재난 | 행안부·기상청 요청 | B2B 담당팀 | 재난대응 ESG 성과 자료 |

---

### 기능 2 — 크리티컬 이슈만 즉시 알림 + 일반 뉴스 주간 묶음

**개념:** 정보 피로도 해소. 중요한 날만 알림이 오도록 설계.

```
S등급 이슈 발생  →  즉시 텔레그램·이메일 발송 (24시간 이내)
A-B등급         →  당일 오전 브리핑에 포함
C-D등급 10건    →  알림 생략, 매주 월요일 주간 요약으로 묶음
이슈 없는 날    →  브리핑 완전 생략
```

**알림 채널:** 텔레그램 Bot API + 이메일 (SMTP)

---

### 기능 3 — 복수 법령 연쇄 리스크 교차 검증 (Cross-RAG)

**개념:** 단일 조문 검색을 넘어, 특정 전파법 조항 변동 시 연쇄적으로 영향받는 타 법령을 한 화면에 매핑.

```
전파법 §43 변경
  → 전파법 §10 (재할당 요건) 영향
  → 전기통신사업법 §18 (인허가 조건) 연동
  → 정보통신망법 §45 (IDC 보안기준) 충돌 가능성 ⚠
```

**생산성 효과:** 법제처에서 3개 법령 탭을 열고 수동 대조하던 작업 자동화.

---

### 기능 4 — 조문 원문 레이어 팝업 (신뢰도 검증)

**개념:** AI 답변에서 인용된 조문 옆 [원문 보기] 버튼 → `document_chunks` 원문 팝업. 법제처 이동 불필요.

**효과:** 응답이 느려도 "원문을 즉시 대조할 수 있다"는 경험으로 신뢰도 유지.

**구현:** 각 법령 인용에 `chunk_id`를 태깅 → 클릭 시 Supabase `document_chunks` 조회 → 팝업 표시

---

### 기능 5 — `qa_knowledge` 팀 검증 Q&A 자동 주입

**개념:** 팀원이 검증 완료한 AI 답변을 `qa_knowledge` 테이블에 저장. 이후 유사 질문 입력 시 Claude API 호출 전에 검증된 답변을 최상단 노출.

```
유사 질문 감지 → qa_knowledge 먼저 조회 → 있으면 즉시 노출 (빠름)
                                          → 없으면 RAG + Claude API 호출
```

**효과:** 응답 속도 향상 + 시니어 노하우가 주니어에게 자연 전수.

**Supabase 테이블:**
```sql
qa_knowledge (
  id, question, answer, category,
  verified_by, verified_at, related_law,
  use_count, is_active
)
```

### 기능 6 — 고시 개정안 자동 Diff 분석

**개념:** 크롤러가 과기정통부·국립전파연구원 입법예고를 포착하면, 기존 DB 조문과 신조문을 나란히 비교(Red/Green). 독소조항 자동 하이라이트.

**출력 예시:**
```
구조문: "이용기간 내 예상 매출의 3%"  (빨간색 취소선)
신조문: "잔여 이용기간 기준 예상 매출의 3%"  (초록색 추가)

⚠ 독소조항 감지: 단기 재할당 시 대가 산정 기준 불리하게 작용 가능
```

**구현:** law.go.kr / rra.go.kr 크롤링 → 기존 `document_chunks` 텍스트 비교 → diff 결과 저장

### 기능 7 — CR 규제 리스크 신호등 Dashboard Widget

**개념:** 대시보드 진입 즉시 S/A-B/C-D 등급별 이슈 건수와 내용을 시각화. 의사결정권자가 텍스트 읽기 전에 현황 파악.

**표시 요소:**
- 🔴 S등급 N건 (즉시 대응 필요 이슈 목록)
- 🟡 A-B등급 N건 (주의 이슈 목록)
- 🟢 C-D등급 N건 (안정, 클릭하면 주간 요약)
- 오늘 팀 액션 아이템 (KPI 연동)

---

### 기능 8 — 국회 과방위 발의안 동향 추적

**개념:** 국회 의안정보시스템에서 과방위 발의안을 수집하여 SKT 비즈니스 영향을 AI가 사전 스크리닝.

**분류:**
- 부정적 영향 (빨간 딱지) → A등급 이슈로 자동 분류
- 중립/긍정 (파란 딱지) → B-C등급

**수집 대상:** `likms.assembly.go.kr` 의안정보시스템 RSS 또는 크롤링

## 5. 시스템 프롬프트 (Claude API용)

```
[역할 정의] SK텔레콤 Comm센터 기술정책팀 전파정책 에이전트 (RegTech)

당신은 SK텔레콤 Comm센터 기술정책팀의 수석 전파정책 전문위원입니다.
이 팀은 이동통신 주파수(할당·재할당·전파사용료·경매·무선국·품질평가),
통신재난(재난문자·재난대응), 기술 규제 이슈를 전담하는 대관 전략 조직입니다.

[2026년 핵심 방어 목표 — 답변 생성 시 항상 고려]
① (35%) 기지국 예비전원 의무화 확대 저지 + 전파사용료 감면 확보
② (35%) 5G 추가 주파수 공급 2027년 이후 지연 + 재할당 대가 인상 저지
③ (15%) SKT 우호적 5G 품질평가 정책 환경 조성
④ (15%) 통신재난 대응 + B2B 정책 지원

[리스크 등급 판정 원칙]
"이 규제 변화가 위 4개 KPI 중 어디에 영향을 미치는가"를 기준으로
S / A / B / C-D 등급을 판정하고, 해당 KPI의 액션 템플릿에 따라
팀 액션 아이템을 강제 도출하라.

[핵심 차별화 행동 규칙]

1. 임원 보고용 '규제 리스크 신호등' 표시
   모든 답변 최상단에 한 줄로 등급 표시:
   "● [규제 리스크] 🟡 주의 (A등급) — [이슈 요약]"

2. 팀 액션 아이템 표 강제 도출
   답변 하단에 [즉시 확인 / 임원 보고 / 유관부서 협의 / 필요 산출물 / 일정 알림] 5개 행 고정

3. 복수 법령 연쇄 리스크 교차 검증 (Cross-RAG)
   단일 조문에서 연쇄 영향 법령까지 한 화면에 매핑

4. 조문 원문 인용 의무
   조항 번호만 나열 금지. 원문 내용 반드시 함께 서술.
   원문 확인 불가 시 "law.go.kr 직접 확인 필요" 명시.

5. 인식론적 구분
   [확정된 사실(법조문 원문)] / [해석 및 추정] / [정책 의견] 구분 표시

6. 공문 대응 모드 (트리거: 공문·발의안 텍스트 입력 시)
   자동 산출: ① 정부 요구사항 요약 ② 임원 보고서 1p 초안 ③ 정부 회신문 초안 ④ 유관부서 협조 메일 초안

7. 독소조항 스크리닝 (트리거: 신구조문 입력 시)
   문구 변경(예: "~할 수 있다" → "~하여야 한다") 자동 감지 및 SKT 불이익 분석

8. qa_knowledge 우선 참조
   RAG 검색 전에 팀 검증 Q&A에서 유사 질문 먼저 확인. 있으면 최상단 노출.
```

## 6. UI 구조 (화면 설계)

### 6-1. 사이드바 메뉴 구성

```
[상황 관제]
  ├── 규제 현황판       ← 메인 홈 (리스크 신호등 + 액션 아이템)
  ├── 규제 레이더       ← 입법예고·고시 개정 타임라인
  └── 과방위 발의 동향  ← 과방위 법안 추적

[AI 자문]

```

## 7. 기술 스택

| 계층 | 기술 | 용도 |
|---|---|---|
| AI 엔진 | Claude API (claude-sonnet-4-6) | 자문, 리스크 분류, 액션 도출 |
| 프론트엔드 | HTML + Vanilla JS + CSS (단일 파일) | 웹 대시보드 |
| 백엔드 DB | Supabase (PostgreSQL) | 뉴스·법령·브리핑·Q&A 저장 |
| 배포 | GitHub Pages | 정적 사이트 호스팅 |
| 자동화 | GitHub Actions (cron) | 크롤링·브리핑 자동 실행 |
| 알림 | 텔레그램 Bot API + SMTP | S등급 즉시 알림 |
| 크롤러 | Python (requests + BeautifulSoup) | 뉴스·입법예고·과방위 수집 |
